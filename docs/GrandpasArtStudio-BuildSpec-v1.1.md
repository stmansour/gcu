# Grandpa's Art Studio — Build Specification
### Version 1.1 — Chapter 1: Color Mixing
### Target Platforms: iPad (primary), iPhone, Desktop (development)

---

## Overview

Grandpa's Art Studio is an interactive color-mixing game for children ages 7–10 (primary target: Kaila, age 9). The player discovers Grandpa's old art studio and learns to mix colors through hands-on experimentation and puzzle-based missions.

This spec covers **Chapter 1: Color Mixing** — the MVP build. It is a single-page HTML5/CSS/JavaScript web application developed in the browser and deployed to iPad and iPhone as a native app via **Capacitor**.

### Platform Strategy

- **Development:** HTML5/CSS/JavaScript in the browser (Chrome/Safari on desktop)
- **Deployment:** Capacitor wraps the web app in a native iOS shell → real App Store app
- **Primary target:** iPad (landscape, touch-first)
- **Secondary target:** iPhone (portrait, touch-first)
- **Tertiary:** Desktop browser (development and fallback)

All code is vanilla HTML/CSS/JavaScript. No React, no frameworks. Capacitor adds the native shell — the game code stays the same across all platforms.

## Visual Design Direction

### Aesthetic: Warm Workshop / Pixar-Quality Storybook

The art studio should feel like a real artist's workshop — warm wood tones, natural light, paint-splattered surfaces. Not sterile or digital-feeling. Think Pixar's art department crossed with a cozy attic studio.

### Color Palette

- **Background/Workshop:** Warm wood tones (#8B6914, #A0784C, #D4A959), soft cream (#FFF8E7)
- **UI Elements:** Soft charcoal (#3A3A3A), warm white (#FFFEF5)
- **Accent/Highlights:** Golden sunlight (#FFD54F), soft teal (#5CBDB9)
- **Paint Colors:** Rich, saturated primaries — not neon, not pastel

### Typography

- Use a warm, hand-drawn or storybook-style display font for headers (e.g., Google Fonts: "Fredoka One", "Baloo 2", or "Lilita One")
- Use a clean, friendly body font for instructions (e.g., "Nunito", "Quicksand")
- All text should be large enough for children to read easily — minimum 18px body, 28px+ headers
- **IMPORTANT:** Bundle font files locally in `assets/fonts/` rather than loading from Google Fonts CDN. The Capacitor app may be used offline, and CDN fonts won't load without connectivity. Download the .woff2 files and use `@font-face` declarations.

- **iPad:** Landscape orientation, three-panel layout left to right:
  1. **Paint Palette** (left ~25%) — paint blobs and mixing area
  2. **Canvas / Challenge Area** (center ~50%) — where painting and color targets appear

## Game Architecture

### State Machine

The game operates as a simple state machine:

```
STATES:
  TITLE_SCREEN    → Player sees studio door, taps to enter
  JOURNAL_INTRO   → Grandpa's journal page appears for current mission
  MISSION_ACTIVE  → Player is mixing/painting to complete a challenge
  MISSION_SUCCESS → Target achieved, celebration animation plays
  SANDBOX_MODE    → Free mixing and painting (unlocked after missions)
```

### Transitions

```
TITLE_SCREEN    → tap "Enter Studio"      → JOURNAL_INTRO (Mission 1)
JOURNAL_INTRO   → tap "Let's Try It!"     → MISSION_ACTIVE
MISSION_ACTIVE  → color match achieved    → MISSION_SUCCESS
MISSION_SUCCESS → tap "Next"              → JOURNAL_INTRO (next mission)
MISSION_SUCCESS → tap "Keep Painting"     → SANDBOX_MODE
MISSION_SUCCESS → (last mission complete) → SANDBOX_MODE (permanent unlock)
SANDBOX_MODE    → tap "Missions"          → JOURNAL_INTRO (next incomplete mission)
```

### Persistent Storage (Save Progress)

Game state must persist between app launches. Use a dual-storage approach:

```javascript
// storage.js — Works in both browser (development) and Capacitor (production)

import { Preferences } from '@capacitor/preferences';

const STORAGE_KEY = 'grandpa_art_studio_state';

// Detect if running inside Capacitor or plain browser
const isCapacitor = typeof window !== 'undefined' && window.Capacitor !== undefined;

export async function saveState(state) {
  const json = JSON.stringify(state);
  if (isCapacitor) {
    await Preferences.set({ key: STORAGE_KEY, value: json });
  } else {
    localStorage.setItem(STORAGE_KEY, json);
  }
}

export async function loadState() {
  let json;
  if (isCapacitor) {
    const result = await Preferences.get({ key: STORAGE_KEY });
    json = result.value;
  } else {
    json = localStorage.getItem(STORAGE_KEY);
  }
  return json ? JSON.parse(json) : null;
}
```

Save state on every mission completion and sandbox entry. Load on app launch to resume where the player left off.

## Core Mechanic: Color Mixing

### How Mixing Works

1. **Paint blobs** sit on the left palette — draggable circles of Red, Blue, Yellow, White, Black.
2. The **mixing well** is a circular area (like a painter's palette depression) in the center-left.
3. Player **drags a paint blob** into the mixing well. The blob animates (splat effect) and the well changes color.
4. Player drags **another blob** in. The color blends with what's already there.
5. Each additional blob shifts the mix. The result is always visible in real-time.
6. A **"Clear Well"** button resets the mixing well to empty.

### Color Mixing Algorithm

Use **subtractive color mixing** (simulating paint, not light). Implement via weighted HSL averaging with the following approach:

```javascript
// Each blob added to the well has equal weight.
// The mixed color is computed from all blobs in the well.

function mixColors(colorBlobs) {
  // colorBlobs: array of {h, s, l} values
  if (colorBlobs.length === 0) return null;
  if (colorBlobs.length === 1) return {...colorBlobs[0]};

  // For hue: use circular mean to handle wraparound (e.g., red=0° and red=360°)
  let sinSum = 0, cosSum = 0;
  let sSum = 0, lSum = 0;

  // Subtractive mixing: lightness darkens as colors combine
  // Each added color contributes to hue/saturation via averaging
  // but lightness uses a subtractive model (gets darker with more pigment)

  for (const c of colorBlobs) {
    const hRad = (c.h * Math.PI) / 180;
    sinSum += Math.sin(hRad);
    cosSum += Math.cos(hRad);
    sSum += c.s;
    lSum += c.l;
  }

  const n = colorBlobs.length;
  let avgH = (Math.atan2(sinSum / n, cosSum / n) * 180) / Math.PI;
  if (avgH < 0) avgH += 360;

  let avgS = sSum / n;

  // Subtractive lightness: more pigments = darker
  // Use geometric-style reduction rather than simple average
  let avgL = lSum / n;
  // Apply darkening factor: each pigment beyond the first darkens slightly
  const darkenFactor = Math.max(0.7, 1 - (n - 1) * 0.08);
  avgL = avgL * darkenFactor;

  return { h: avgH, s: Math.min(100, avgS), l: Math.max(5, Math.min(95, avgL)) };
}
```

**Special case overrides** to make mixing feel natural for kids:

| Mix | Expected Result | Override Needed |
|-----|----------------|-----------------|
| Red + Blue | Purple (~270°) | May need hue nudge since red=0° and blue=220° circular mean can be off |
| Red + Yellow | Orange (~30°) | Should work naturally |
| Blue + Yellow | Green (~140°) | Circular mean of 220° and 48° needs correction — force toward 140° |
| Red + Blue + Yellow | Brown/dark (~30°, low sat, low light) | Desaturate + darken significantly |
| Any + White | Lighter version | Only affects lightness upward |
| Any + Black | Darker version | Only affects lightness downward |

Implement a lookup table for known primary-pair mixes to override the algorithm when only two primary colors are present, ensuring kids get the "right" result (blue + yellow = green, not teal).

### Color Comparison (Match Detection)

```javascript
function colorsMatch(mixed, target, tolerance = 15) {
  // Compare hue (circular distance), saturation, and lightness
  const hueDiff = Math.min(
    Math.abs(mixed.h - target.h),
    360 - Math.abs(mixed.h - target.h)
  );
  const satDiff = Math.abs(mixed.s - target.s);
  const lightDiff = Math.abs(mixed.l - target.l);

  // Weighted comparison — hue matters most, then saturation, then lightness
  const score = (hueDiff * 1.0) + (satDiff * 0.5) + (lightDiff * 0.5);

  return {
    isMatch: score <= tolerance,
    score: score,
    proximity: Math.max(0, 1 - score / (tolerance * 2))
    // proximity: 0 = far away, 1 = perfect match
  };
}
```

### Proximity Feedback ("Hot and Cold")

As the player mixes, show real-time visual feedback:

- A **proximity indicator** near the target swatch — a small ring/glow around the target.
- **Far away:** Ring is dim/grey.
- **Getting warm:** Ring starts glowing softly in warm yellow.
- **Very close:** Ring pulses brighter.
- **Match:** Ring bursts with sparkle animation + success sound.

Do NOT show numbers or percentages. The feedback is purely visual/emotional.

## Mission Structure — Chapter 1: Color Mixing

### Mission 1: "Discovery" — Mix Your First Color

**Journal Page:**
*"I left some paints out on the palette. I wonder what happens if you mix two of them together? Try dragging red and blue into the mixing well and see what you get. — Grandpa"*

**Objective:** Drag any two primary colors into the mixing well.

**Success Condition:** Any two primary blobs placed in well. (Cannot fail — pure discovery.)

**On Success:** The mixed color appears with a gentle sparkle. Text: "You made [color name]! Every new color starts with just two paints."

**Purpose:** Teach the drag-to-mix interaction. Zero frustration.

### Mission 2: "Purple Rain" — Match a Target

**Journal Page:**
*"Can you make the same purple I used for the night sky in my painting? I mixed red and blue, but I don't remember exactly how much of each. Your turn to figure it out! — Grandpa"*

**Objective:** Match a target purple swatch (h: 270, s: 70, l: 40).

**Success Condition:** Mixed color within tolerance of target (score ≤ 15).

**Available Paints:** Red, Blue (only these two active for this mission — keeps it focused).

**Hint (via proximity indicator):** Ring glows warmer as she approaches the target.

**On Success:** The target swatch and mixed swatch merge with a satisfying snap. A small portion of a painting (night sky) colorizes from grey to the mixed purple.

**Objective:** Match a target orange (h: 25, s: 90, l: 55).

**Success Condition:** Mixed color within tolerance.

**Available Paints:** Red, Yellow.

**On Success:** Another section of the painting colorizes — the sunset horizon glows orange.

### Mission 4: "Forest Floor" — The Tricky One

**Journal Page:**
*"Green is the hardest color to get right. There are a hundred different greens in a forest. This one — the deep green of moss on a rock — took me three tries. Don't worry if it takes you a few tries too. — Grandpa"*

**Journal Page:**

**Success Condition:** Mixed color within tolerance.

**Available Paints:** Blue, White (introducing White for lightening).

**Key Teaching Moment:** She learns that adding white lightens a color. Mirrors Mission 4's lesson with black.

### Mission 6: "The Complete Palette" — Multi-Target Challenge

## Touch / Input Handling (iOS-First)

### CRITICAL: Touch Event Architecture

iOS touch handling has specific requirements. Do NOT use HTML5 Drag and Drop API — it does not work on iOS Safari or WKWebView. Implement custom touch handling.

```javascript
// dragDrop.js — iOS-compatible drag and drop

class DragDropHandler {
  constructor(container) {
    this.container = container;
    this.activeBlob = null;
    this.clone = null;

    // Bind touch AND mouse events (mouse for desktop development)
    container.addEventListener('touchstart', this.onStart.bind(this), { passive: false });
    container.addEventListener('touchmove', this.onMove.bind(this), { passive: false });
    container.addEventListener('touchend', this.onEnd.bind(this), { passive: false });

    // Mouse fallback for desktop development
    container.addEventListener('mousedown', this.onStart.bind(this));
    container.addEventListener('mousemove', this.onMove.bind(this));
    container.addEventListener('mouseup', this.onEnd.bind(this));
  }

  getPoint(e) {
    // Normalize touch vs mouse coordinates
    if (e.touches && e.touches.length > 0) {
      return { x: e.touches[0].clientX, y: e.touches[0].clientY };
    }
    return { x: e.clientX, y: e.clientY };
  }

  onStart(e) {
    const target = e.target.closest('.paint-blob');
    if (!target) return;

    e.preventDefault();  // CRITICAL: prevents iOS scroll/zoom during drag

    const point = this.getPoint(e);
    this.activeBlob = target;

    // Create visual clone that follows finger
    this.clone = target.cloneNode(true);
    this.clone.classList.add('dragging');
    this.clone.style.position = 'fixed';
    this.clone.style.zIndex = '9999';
    this.clone.style.pointerEvents = 'none';
    this.clone.style.transform = 'scale(1.1)';
    this.updateClonePosition(point);
    document.body.appendChild(this.clone);
  }

  onMove(e) {
    if (!this.clone) return;

    e.preventDefault();  // CRITICAL: prevents iOS rubber-banding during drag

    const point = this.getPoint(e);
    this.updateClonePosition(point);

    // Optional: show visual feedback when hovering over mixing well
    const overWell = this.isOverMixingWell(point.x, point.y);
    this.clone.style.transform = overWell ? 'scale(1.2)' : 'scale(1.1)';
  }

  onEnd(e) {
    if (!this.clone) return;

    const point = e.changedTouches
      ? { x: e.changedTouches[0].clientX, y: e.changedTouches[0].clientY }
      : { x: e.clientX, y: e.clientY };

    if (this.isOverMixingWell(point.x, point.y)) {
      // Trigger splat animation + add color to mix
      this.onDropInWell(this.activeBlob.dataset.color);
    } else {
      // Fade out clone
      this.clone.style.transition = 'opacity 0.3s';
      this.clone.style.opacity = '0';
    }

    // Cleanup
    setTimeout(() => {
      if (this.clone && this.clone.parentNode) {
        this.clone.parentNode.removeChild(this.clone);
      }
      this.clone = null;
      this.activeBlob = null;
    }, 300);
  }

  updateClonePosition(point) {
    // Center the clone on the touch point
    const size = this.clone.offsetWidth;
    this.clone.style.left = (point.x - size / 2) + 'px';
    this.clone.style.top = (point.y - size / 2) + 'px';
  }

  isOverMixingWell(x, y) {
    const well = document.querySelector('.mixing-well');
    if (!well) return false;
    const rect = well.getBoundingClientRect();
    const centerX = rect.left + rect.width / 2;
    const centerY = rect.top + rect.height / 2;
    const radius = rect.width / 2;
    const dist = Math.sqrt((x - centerX) ** 2 + (y - centerY) ** 2);
    return dist <= radius * 1.3;  // Generous forgiveness zone for kids
  }

  onDropInWell(colorName) {
    // Implemented by app.js — add color to mix, trigger animation
  }
}
```

### Key iOS Touch Rules

1. **Always use `{ passive: false }` on touchstart/touchmove** when you call `preventDefault()`. iOS requires this explicitly.
2. **Always `preventDefault()` on touchstart and touchmove** for drag interactions — otherwise iOS will try to scroll, zoom, or rubber-band.
3. **Use `touchend` `e.changedTouches` not `e.touches`** — by the time touchend fires, `e.touches` is empty (the finger lifted).
4. **No `:hover` styles for interactive elements.** iOS "sticky hover" will make things look broken. Use `:active` instead, or only apply hover in a `@media (hover: hover)` query.
5. **Touch targets must be at least 44px × 44px** (Apple HIG minimum). For children, use 60px+.
6. **Set `touch-action: none`** on draggable elements and the mixing well to prevent the browser from intercepting the gesture.

```css
.paint-blob,
.mixing-well {
  touch-action: none;   /* Prevent browser from hijacking touch for scroll/zoom */
}

/* Only apply hover effects on devices that actually support hover */
@media (hover: hover) {
  .paint-blob:hover {
    transform: scale(1.05);
    transition: transform 0.2s ease;
  }
}

/* Active state works on both touch and mouse */
.paint-blob:active {
  transform: scale(0.95);
}
```

### Canvas Painting (Sandbox) — iOS Specifics

```javascript
// canvas.js — iOS-compatible painting

class PaintCanvas {
  constructor(canvasElement) {
    this.canvas = canvasElement;
    this.ctx = canvasElement.getContext('2d');
    this.painting = false;
    this.lastPoint = null;

    // CRITICAL: Set canvas resolution to match device pixel ratio
    // Without this, canvas looks blurry on Retina iPad/iPhone displays
    this.setupRetina();

    // Touch events
    this.canvas.addEventListener('touchstart', this.startStroke.bind(this), { passive: false });
    this.canvas.addEventListener('touchmove', this.continueStroke.bind(this), { passive: false });
    this.canvas.addEventListener('touchend', this.endStroke.bind(this));

    // Mouse fallback
    this.canvas.addEventListener('mousedown', this.startStroke.bind(this));
    this.canvas.addEventListener('mousemove', this.continueStroke.bind(this));
    this.canvas.addEventListener('mouseup', this.endStroke.bind(this));
  }

  setupRetina() {
    // iPad Pro: devicePixelRatio = 2
    // iPhone: devicePixelRatio = 2 or 3
    const dpr = window.devicePixelRatio || 1;
    const rect = this.canvas.getBoundingClientRect();

    // Set internal resolution to match physical pixels
    this.canvas.width = rect.width * dpr;
    this.canvas.height = rect.height * dpr;

    // Scale context so drawing coordinates match CSS coordinates
    this.ctx.scale(dpr, dpr);

    // Keep CSS size unchanged
    this.canvas.style.width = rect.width + 'px';
    this.canvas.style.height = rect.height + 'px';
  }

  getPoint(e) {
    const rect = this.canvas.getBoundingClientRect();
    if (e.touches && e.touches.length > 0) {
      return {
        x: e.touches[0].clientX - rect.left,
        y: e.touches[0].clientY - rect.top
      };
    }
    return { x: e.clientX - rect.left, y: e.clientY - rect.top };
  }

  startStroke(e) {
    e.preventDefault();
    this.painting = true;
    this.lastPoint = this.getPoint(e);
    this.drawDot(this.lastPoint);
  }

  continueStroke(e) {
    if (!this.painting) return;
    e.preventDefault();
    const point = this.getPoint(e);

    // Interpolate between last point and current point for smooth strokes
    this.drawLine(this.lastPoint, point);
    this.lastPoint = point;
  }

  endStroke(e) {
    this.painting = false;
    this.lastPoint = null;
    // Save canvas state for undo (store as ImageData)
    this.saveUndoState();
  }

  drawDot(point) {
    this.ctx.beginPath();
    this.ctx.arc(point.x, point.y, this.brushSize, 0, Math.PI * 2);
    this.ctx.fillStyle = this.currentColor;
    this.ctx.fill();
  }

  drawLine(from, to) {
    this.ctx.beginPath();
    this.ctx.moveTo(from.x, from.y);
    this.ctx.lineTo(to.x, to.y);
    this.ctx.strokeStyle = this.currentColor;
    this.ctx.lineWidth = this.brushSize * 2;
    this.ctx.lineCap = 'round';
    this.ctx.lineJoin = 'round';
    this.ctx.stroke();
  }
}
```

## Animations & Feedback

### Paint Splat (on drop into well)
- Blob scales to 1.3x, squishes to oval, rebounds to circle
- Small colored particles spray outward (6–8 dots, CSS animations)
- Duration: 400ms
- **Use CSS animations, not JavaScript animation loops** — CSS animations are GPU-accelerated on iOS and won't cause jank

### Color Swirl (in mixing well)
- Brief circular gradient rotation in the well
- Duration: 600ms, ease-out

### Proximity Glow (around target swatch)
- Continuous soft pulse when close (opacity 0.3 → 0.7, 1.5s loop)
- Intensity increases with proximity score
- Color shifts from cool grey (far) to warm gold (close)

### Match Success
- Target ring bursts outward (scale 1.0 → 1.5, fade out)
- Sparkle particles (CSS stars, 8–12 pieces)
- Satisfying snap/pop sound effect (see Audio section)
- Both swatches glow and merge
- Duration: 800ms

### Painting Section Reveal
- Greyscale section crossfades to full color
- Slight scale pulse (1.0 → 1.02 → 1.0) to draw attention
- Duration: 1200ms

### Mission Complete Celebration (Mission 6 only)
- Paint splashes animate across screen from edges
- Painting fully illuminates
- Confetti-style colored dots rain down
- Duration: 2500ms

### iOS Animation Best Practices
- **Use `transform` and `opacity` only** for animations where possible — these are the only CSS properties iOS can animate on the GPU without repainting.
- **Use `will-change: transform`** on elements that animate frequently (paint blobs, mixing well).
- **Avoid animating `box-shadow`, `border-radius`, `filter`** during active gameplay — these trigger CPU repaints on iOS.
- **Use `requestAnimationFrame`** for any JavaScript-driven animation, never `setInterval`.

```css
/* GPU-accelerated animation example */
.paint-blob {
  will-change: transform;
  transition: transform 0.2s ease;
}

.splat-particle {
  will-change: transform, opacity;
  animation: splat-fly 0.4s ease-out forwards;
}

@keyframes splat-fly {
  0% { transform: scale(0) translate(0, 0); opacity: 1; }
  100% { transform: scale(1) translate(var(--dx), var(--dy)); opacity: 0; }
}
```

## Audio

### iOS Audio Context Requirement

**CRITICAL:** iOS Safari and WKWebView require a user gesture to unlock audio playback. You cannot play sounds programmatically until the user has tapped something.

```javascript
// audio.js — iOS-compatible audio manager

class AudioManager {
  constructor() {
    this.audioContext = null;
    this.sounds = {};
    this.unlocked = false;
    this.muted = false;
  }

  // Call this on the FIRST user tap (e.g., "Enter Studio" button)
  unlock() {
    if (this.unlocked) return;

    // Create AudioContext on first user gesture
    this.audioContext = new (window.AudioContext || window.webkitAudioContext)();

    // iOS requires playing a silent buffer to fully unlock
    const buffer = this.audioContext.createBuffer(1, 1, 22050);
    const source = this.audioContext.createBufferSource();
    source.buffer = buffer;
    source.connect(this.audioContext.destination);
    source.start(0);

    this.unlocked = true;
  }

  async loadSound(name, url) {
    if (!this.audioContext) return;
    const response = await fetch(url);
    const arrayBuffer = await response.arrayBuffer();
    this.sounds[name] = await this.audioContext.decodeAudioData(arrayBuffer);
  }

  play(name) {
    if (this.muted || !this.unlocked || !this.sounds[name]) return;
    const source = this.audioContext.createBufferSource();
    source.buffer = this.sounds[name];
    source.connect(this.audioContext.destination);
    source.start(0);
  }
}
```

### Sound Effects (keep files small — all under 50KB each)

- **Paint splat:** Soft wet splat sound on drop (~0.3s)
- **Color swirl:** Gentle liquid swoosh (~0.5s)
- **Match success:** Bright chime/ding (~0.4s)
- **Mission complete:** Short musical flourish (~1.5s)
- **Painting reveal:** Gentle shimmer/sparkle (~1s)

Store as `.mp3` files in `assets/audio/`. MP3 is universally supported in iOS WKWebView.

All audio should be optional and mutable via a speaker icon toggle (persisted in game state). The game must work fully without sound.

## Performance Guidelines for iOS

1. **DOM node count:** Keep under 200 visible nodes. iPad Safari slows down with excessive DOM.
2. **Canvas resolution:** Match `devicePixelRatio` (see Retina setup above) but don't exceed it — a 2x canvas on iPad Pro is already 2048×1536 pixels.
3. **Image assets:** Use SVG for the painting and UI elements (scales perfectly, small file size). Use compressed PNG only for textures (journal background).
4. **Font loading:** Preload fonts before showing game content to avoid FOUT (flash of unstyled text):
   ```html
   <link rel="preload" href="assets/fonts/fredoka-one.woff2" as="font" type="font/woff2" crossorigin>
   ```
5. **Memory:** Undo history in sandbox should be limited (max 10 states). Each canvas `ImageData` snapshot on a 2x Retina iPad is ~12MB in memory.
6. **No `setInterval` for animations.** Always use `requestAnimationFrame` or CSS animations.
